﻿# fluxsec.dev

Repositorio de estudio y laboratorio de ciberseguridad.

Este proyecto organiza los contenidos por áreas temáticas (blue team, DFIR, pentesting, criptografía, etc.) para documentar teoría, prácticas y laboratorios, y mantener trazabilidad de todo lo trabajado.

## Estructura general

- \lue-team/\ — Detección, respuesta a incidentes, playbooks y herramientas defensivas.
- \criptografia/\ — Criptografía simétrica, asimétrica, hashing, PKI y ejercicios.
- \devsecops/\ — Seguridad en el ciclo CI/CD, análisis de código, contenedores y Kubernetes.
- \dfir/\ — Digital Forensics & Incident Response: análisis forense, timeline, herramientas y labs.
- \undamentos/\ — Fundamentos de redes, sistemas, modelos de amenaza y conceptos básicos.
- \
etworking/\ — Capas OSI, protocolos, laboratorios de red, Wireshark y comandos.
- \pentesting/\ — Reconocimiento, explotación, post-explotación, herramientas y labs.
- \soc/\ — Monitorización, casos de uso, SIEM, alertas y procedimientos.
- \malware/\ — Análisis estático y dinámico, sandbox, firmas y laboratorios.
- \linux/\ — Comandos, servicios, seguridad, scripts y laboratorios.
- \windows/\ — Comandos, Active Directory, seguridad, GPO y labs.
- \cloud/\ — Seguridad en AWS, Azure, GCP, infra as code y arquitectura cloud.
- \proyecto/\ — Planificación, tareas, infraestructura y entregables de proyectos propios.
- \documentacion/\ — Plantillas, diagramas, imágenes y referencias generales.

Cada carpeta temática contiene subcarpetas específicas como \labs\, \documentacion\, \herramientas\, etc., para separar claramente teoría, práctica y recursos.

## Convenciones

- Se usa \.gitkeep\ para asegurar que Git mantenga la estructura de carpetas.
- Toda la documentación se redacta en formato Markdown.
- Los laboratorios y ejercicios prácticos deben tener pasos claros y reproducibles.
